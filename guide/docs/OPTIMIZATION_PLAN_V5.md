# 零一导购 V5：资产库与 AI 剪辑工作台优化计划

> 状态：需求基线  
> 更新时间：2026-06-23  
> 范围：资产库、编辑器、分镜流程、AI 素材生产、预览一致性  
> 约束：保留“资产库 + 模板 DSL + `template_editor` + Stage4 FFmpeg”主干，不新增用户可见流水线。

## 1. 目标与判断

V5 不再继续堆叠资产类型、面板和 AI 按钮。用户侧主路径固定为：

`选择模板或输入商品 → 形成可编辑分镜 → 补齐素材 → 预览 → 生成视频`

当前主要问题：

1. 资产库按技术对象拆分过细，用户需要先理解系统数据模型。
2. 编辑器同时出现分镜、脚本、画布、属性和生成步骤，重复表达同一内容。
3. 预览只能证明“有内容”，不能回答成片构图、动效、字幕安全区和前后版本差异。
4. AI 能力以独立按钮存在，缺少明确作用范围、输入引用、生成版本和成本预期。
5. 资产数量不足时主要表现为空状态，没有用模板资源、品牌派生素材和 AI 补全形成生产闭环。

V5 的核心对象由“资产类型 + 时间轴元素”调整为“镜头 Shot”。资产、脚本、数字人、字幕、动效和生成记录都围绕镜头聚合；时间轴保留为精调工具，不承担首次成片的主流程。

## 2. 调研结论与可复用边界

### 2.1 MkBoard 0.1.11（本地应用实测）

MkBoard 的工作流是“脚本 → AI 解析分镜 → 分镜表 → 图片/视频生成 → 演示或视频”，不是传统多轨剪辑器。

值得提取：

- 项目创建时一次选择“分镜技能 + 视频风格”，后续作为生产约束，不在每个镜头重复配置。
- 分镜表字段可配置，镜头可排序；AI 重解析会明确提示覆盖范围。
- 图片与视频生成分开，支持只生成缺失项，避免无意义重算。
- 技能、风格、模板均使用可视化卡片；风格库以大缩略图表达结果预期。
- Agent 在执行前读取脚本、分镜表、项目素材和技能，过程状态对用户可见。

不直接复制：MkBoard 应用代码、插画、模板卡片和内置风格素材。产品未提供可复用开源许可，只提取交互机制并重新设计实现。

### 2.2 Palmier Pro

值得提取：

- AI 命令绑定当前素材、当前镜头或选中时间范围，而不是全局模糊执行。
- 素材生成记录提示词、模型、时长、画幅、参考素材和成本。
- 可折叠面板和工作区预设，让新手与精调用户共享同一编辑器。
- Agent 先 `inspect` 再执行，批量操作合并为一个撤销步骤。

许可边界：GPLv3。V5 只借机制与信息架构，不复制实现代码到当前产品。

### 2.3 diffusionstudio/lottie

值得提取：

- 多场景动效包、`slots` 可编辑字段和 `controls.json` 控制定义。
- `?frame=N` 精确帧预览，以及首帧/中间帧/末帧三点校验。
- 用户改动写回单一源数据，预览与导出读取同一配置。

许可边界：MIT。可在保留版权声明后评估代码级复用；首期仍采用“HF 预览 + 透明 WebM/PNG 序列 + FFmpeg overlay”，不引入第二成片引擎。

### 2.4 GSAP UI

只用于界面反馈和预览层：列表重排过渡、拖拽吸附、面板切换、字幕拆分演示。业务状态、时间轴数据和最终成片不能依赖 GSAP。引入前单独锁定版本并复核官方许可。

### 2.5 OpenCut Classic 与 DesignCombo

- OpenCut Classic：MIT。可评估媒体拖入、网格/列表切换、排序、素材高亮定位、贴纸/音效/文字/转场资源面板的实现。
- OpenCut 新版：仍在重写，当前只跟踪其 Editor API、插件优先、MCP、Headless 和脚本页方向，不作为 V5 依赖。
- DesignCombo React Video Editor：仓库未声明开源许可证。只参考实时预览、可缩放面板和媒体菜单的信息架构，不复制代码或示例素材。

## 3. V5 信息架构

### 3.1 一级导航

主产品只保留：

1. 项目
2. 资产库
3. 模板
4. 成片

数字人训练、供应商、流水线、任务诊断进入管理或调试页面，不占用导购主路径。

### 3.2 资产库由八类收敛为四组

| 用户分组 | 包含内容 | 核心动作 |
|---|---|---|
| 品牌与角色 | 品牌包、Logo、字体、数字人、声音 | 设为项目默认、查看使用范围 |
| 商品与场景 | 商品图、场景图、视频、B-roll、参考图 | 拖入镜头、AI 衍生、查看版本 |
| 文案与音频 | 脚本、知识、口播、BGM、音效 | 生成脚本、试听、绑定镜头 |
| 模板与动效 | 视频模板、镜头模板、字幕样式、转场、Lottie 贴纸、外观预设 | 预览、应用、收藏 |

类型仍保留在数据层，用筛选器表达，不继续作为八个同级主 Tab。

### 3.3 资产的三种作用域

- 企业资产：品牌、数字人、声音、审核通过模板，可跨项目使用。
- 项目资产：当前商品和本次生产素材，默认只在当前项目展示。
- 镜头生成物：AI 生成版本、临时参考和中间结果，必须归属镜头并记录血缘。

默认先展示当前项目，用户可切换“企业共享”。这样兼顾 MkBoard 的简单性与企业资产复用。

## 4. 资产库交互要求

### 4.1 首页结构

- 顶部：搜索、当前项目/企业共享切换、上传、AI 生产。
- 左侧：四个用户分组及收藏、最近使用、审核状态。
- 中间：大缩略图网格为默认；列表视图用于批量治理。
- 右侧：选中资产的预览、元数据、使用范围、版本和 AI 动作。
- 无选中项时右侧不显示空表单，改为“本项目缺少什么”的生产建议。

### 4.2 空状态必须可生产

空库不能只有“上传素材”，至少提供：

- 从企业资产选择
- 上传商品或参考素材
- 从内置可商用资源创建
- AI 生成商品场景/B-roll
- 应用镜头模板或动效包

生成前显示数量、预计耗时、积分或成本和写入位置。

### 4.3 资产卡片

卡片必须展示缩略图、类型、时长/画幅、来源、审核状态和使用次数。悬停只保留四个高频动作：预览、加入当前镜头、AI 衍生、更多。

AI 生成资产必须记录：

- `parent_asset_ids`
- `project_id` / `segment_id`
- prompt、model、seed、aspect_ratio、duration
- generation status、cost、created_by
- review status、usage rights、provider terms snapshot

禁止临时 URL 直接进入成片。

### 4.4 可视化预览

按资产类型提供正确的预览，而不是统一详情表：

- 图片/视频：适配、裁切、安全区、镜头内实际构图。
- 音频/声音：波形、试听、时长和字幕词轴。
- 品牌包：Logo、色板、字体、字幕、片头/片尾组合板。
- 镜头模板：9:16 实际画布、首中末三帧、可替换槽位。
- Lottie/贴纸：循环预览、精确帧、透明背景和安全区。
- 脚本：镜头数、字数、预估口播时长和品牌词检查。

## 5. 编辑器交互要求

### 5.1 默认工作区：分镜优先

编辑器默认布局：

- 左侧：镜头列表，展示缩略图、口播摘要、时长和完成状态。
- 中央：足够大的实际比例画布；支持源素材、画布、前后对比三种预览。
- 右侧：只显示当前选中镜头或元素的“内容 / 样式 / 动效”。
- 底部：默认是可拖高的脚本面板；时间轴通过“精调”进入，不长期占据空间。

删除重复的场景列表与脚本镜头列表。镜头是唯一导航对象。

### 5.2 镜头 Shot 数据模型

每个镜头最少包含：

```ts
type Shot = {
  id: string
  order: number
  frame_template_id: string
  narration_text: string
  estimated_duration_ms: number
  asset_slots: AssetSlot[]
  variants: ShotVariant[]
  motion_preset_id?: string
  preview: { poster_url?: string; preview_url?: string; status: string }
  status: "missing" | "ready" | "generating" | "warning"
}
```

AI 分镜只允许选择 `frame_template_id` 和填充变量，不允许自由发明未知布局。

### 5.3 AI 操作必须有作用范围

统一 AI 入口，根据选中对象显示动作：

- 资产：去背景、扩图、生成同款、生成场景、生成视频。
- 镜头：精简口播、补素材、换构图、生成变体、重做本镜。
- 时间范围：调整节奏、加 B-roll、处理字幕或音频。

执行前显示“将读取什么、修改什么、生成几个结果、预计成本”。结果先成为 Variant，用户采用后再写回主镜头；整组操作只产生一个撤销点。

### 5.4 预览模式

- 源素材：检查原文件和可用区域。
- 画布预览：HF/Lottie 与 DSL 同源，支持逐镜 seek。
- 前后对比：对比当前版本与 AI Variant，支持滑杆或并排。
- 成片抽样：生成前查看关键镜头首中末三帧；生成后展示边界 QA 和降级提示。

预览必须展示 9:16 安全区、字幕占位、数字人遮挡、Logo/价格/CTA 冲突。

### 5.5 生成流程

生成按钮只打开单行确认：

`共 N 镜 · 约 M 秒 · AI 补 X 项 · 预计 T 分钟/成本 C · 生成`

后台进度映射为用户语言：分析与分镜 → 补齐素材 → 生成口播与数字人 → 合成与质检。失败定位到具体镜头，并提供“修复本镜后继续”，不要求整片重跑。

## 6. 素材资源建设

### 6.1 首批资源包

资源不是简单堆文件，必须是带预览、参数和使用边界的可组合包：

1. 12 套企业导购镜头模板：开场 Hook、商品展示、卖点解释、对比、信任背书、门店/服务、优惠、CTA。
2. 8 套字幕系统：品牌简洁、促销高亮、逐词强调、价格牌、参数卡、问答、步骤、免责声明。
3. 20 个轻量动效：价格 burst、标签、箭头、评分、限时、门店定位、扫码、下单 CTA。
4. 12 个转场与镜头运动预设：只保留能稳定映射到 FFmpeg/HF 的集合。
5. 24 个常用音效与 8 个 BGM 情绪分类；所有资源必须有明确商用许可或为自制。
6. 4 套企业品牌示例包，用于说明结构，不使用第三方真实品牌商标。

### 6.2 来源和合规

- 优先：团队自制、MIT/CC0/明确商用授权资源。
- 引入时保存 `source_url`、作者、许可证、版本、下载日期、允许用途和归属要求。
- GPL 项目不复制实现代码；无许可证仓库和 MkBoard 专有素材不进入产品。
- 示例 CDN 只用于调研，发布资源必须落入本方对象存储和资产表。

## 7. 实施顺序

### P0：流程收敛与预览可信（2 个迭代）

1. 建立 Shot 聚合模型和镜头完成状态。
2. 资产库四组信息架构 + 企业/项目作用域。
3. 编辑器改为分镜优先，移除重复镜头导航。
4. 画布放大，补齐安全区、字幕和遮挡预览。
5. AI 操作增加作用范围、成本预估、Variant 和单步撤销。
6. 生成前缺槽检测和逐镜失败恢复。

### P1：资源丰富与高还原（2 个迭代）

1. 镜头模板、字幕、动效、转场和音频首批资源包。
2. Lottie `slots`、精确帧和首中末三帧预览。
3. 资产血缘、版本对比、审核状态和使用范围。
4. 唇形后词轴重对齐、品牌词校对、Stage4 边界 QA。
5. 源素材/画布/对比三种预览模式。

### P2：AI 原生生产（后续）

1. 商品链接/主题先生成提案，再进入可编辑分镜。
2. Agent inspect-first 工具协议和镜头级批量操作。
3. 参考片节奏分析，只写入建议，不复制内容。
4. 插件或 MCP 接口支持外部工具读取项目、素材和分镜。

## 8. 验收指标

- 新用户从项目创建到首版可预览分镜不超过 5 分钟。
- 80% 以上项目无需打开多轨时间轴即可生成首版。
- 用户在生成前能识别缺失素材、总时长、成本和高风险镜头。
- 资产加入镜头不超过 2 次操作；AI 衍生结果可在原位比较和采用。
- 预览和成片的布局、字幕安全区、动效时序达到模板级一致。
- 单镜失败不重跑整片；所有 AI 生成物可追溯到输入、模型和采用版本。

## 9. 明确不做

1. 不把七条 `pipeline_key` 暴露给主产品用户。
2. 不把 MkBoard、Palmier、OpenCut 或 DesignCombo 的完整产品形态搬入项目。
3. 不以时间轴作为新手主入口。
4. 不让 AI 生成物绕过资产库和审核状态。
5. 不新增 HF/Remotion 第二条默认交付路径。
6. 不复制来源不明、无商用许可或第三方产品内置的素材。
